# Deployment Guide

This guide walks through deploying two scaffolded Azure Functions projects generated by `azure-functions-scaffold` v0.3.2:

- `http` template (HTTP API)
- `timer` template (scheduled background job)

It includes scaffold verification (dry run), Azure resource provisioning, publish commands, endpoint/log verification, and cleanup. Outputs are representative examples, not guaranteed byte-for-byte.

## Prerequisites

| Requirement | Minimum | Notes |
|---|---|---|
| Azure subscription | Active | Use `<YOUR_SUBSCRIPTION_ID>` |
| Azure CLI (`az`) | Current | `az --version` |
| Azure Functions Core Tools (`func`) | v4 | `func --version` |
| Python | 3.10+ | Runtime examples below use Python 3.11 |
| pip | Installable | Used to install scaffold CLI |

## Install the scaffold CLI

```bash
python -m pip install --upgrade pip
pip install azure-functions-scaffold
afs --version
```

Representative output:

```text
Requirement already satisfied: pip in ./.venv/lib/python3.11/site-packages (25.0)
Collecting azure-functions-scaffold
Successfully installed azure-functions-scaffold-0.3.2
afs, version 0.3.2
```

The CLI command is `afs` (alias of `azure-functions-scaffold`) and is built with Typer.

## Example 1: HTTP API (`http` template)

### Scaffold project (dry run)

```bash
afs new my-http-api --template http --dry-run
```

Representative output:

```text
Dry run: create project at <CURRENT_DIR>/my-http-api
Template: http
Preset: standard
Python: 3.10
Files:
  - .funcignore
  - .gitignore
  - Makefile
  - README.md
  - app/__init__.py
  - app/core/__init__.py
  - app/core/logging.py
  - app/functions/__init__.py
  - app/functions/http.py
  - app/schemas/__init__.py
  - app/schemas/request_models.py
  - app/services/__init__.py
  - app/services/hello_service.py
  - function_app.py
  - host.json
  - local.settings.json.example
  - pyproject.toml
  - tests/test_http.py
```

Create the actual project:

```bash
afs new my-http-api --template http
```

> Optional template flags like `--with-openapi` and `--with-validation` are available, but this guide uses the base `http` template only.

### Prepare shell variables

```bash
export SUBSCRIPTION_ID="<YOUR_SUBSCRIPTION_ID>"
export LOCATION="eastus"
export HTTP_RG="<YOUR_HTTP_RESOURCE_GROUP>"
export HTTP_STORAGE="<YOUR_HTTP_STORAGE_ACCOUNT>"
export HTTP_APP="<YOUR_HTTP_FUNCTION_APP_NAME>"
```

### Set active subscription

```bash
az account set --subscription "$SUBSCRIPTION_ID"
```

### Create resource group

```bash
az group create --name "$HTTP_RG" --location "$LOCATION"
```

Representative output:

```json
{"name":"<YOUR_HTTP_RESOURCE_GROUP>","location":"eastus","properties":{"provisioningState":"Succeeded"}}
```

### Create storage account

```bash
az storage account create \
  --name "$HTTP_STORAGE" \
  --resource-group "$HTTP_RG" \
  --location "$LOCATION" \
  --sku Standard_LRS \
  --kind StorageV2
```

Representative output:

```json
{"name":"<YOUR_HTTP_STORAGE_ACCOUNT>","kind":"StorageV2","provisioningState":"Succeeded"}
```

### Create Function App

```bash
az functionapp create \
  --name "$HTTP_APP" \
  --resource-group "$HTTP_RG" \
  --storage-account "$HTTP_STORAGE" \
  --consumption-plan-location "$LOCATION" \
  --runtime python \
  --runtime-version 3.11 \
  --functions-version 4
```

Representative output:

```json
{"name":"<YOUR_HTTP_FUNCTION_APP_NAME>","defaultHostName":"<YOUR_HTTP_FUNCTION_APP_NAME>.azurewebsites.net","provisioningState":"Succeeded","state":"Running"}
```

### Publish the scaffolded HTTP project

```bash
cd my-http-api
func azure functionapp publish "$HTTP_APP"
```

Representative output:

```text
Getting site publishing info...
[2026-03-18T12:00:22.101Z] Starting the function app deployment...
Uploading package...
Uploading 2.18 MB [#############################################################################]
Deployment completed successfully.
Syncing triggers...
Functions in <YOUR_HTTP_FUNCTION_APP_NAME>:
    hello - [httpTrigger]
        Invoke url: https://<YOUR_HTTP_FUNCTION_APP_NAME>.azurewebsites.net/api/hello
Deployment successful.
```

### Verify HTTP endpoint

```bash
export HTTP_BASE_URL="https://$HTTP_APP.azurewebsites.net"
curl -i "$HTTP_BASE_URL/api/hello?name=Azure"
```

Representative response:

```text
HTTP/1.1 200 OK
Content-Type: text/plain; charset=utf-8
Date: Wed, 18 Mar 2026 12:03:41 GMT
Content-Length: 13

Hello, Azure!
```

The hello service uses `build_greeting(name)` and returns plain text:

- With `name=Azure` -> `Hello, Azure!`
- Without `name` -> `Hello, world!`

Verify default behavior:

```bash
curl -i "$HTTP_BASE_URL/api/hello"
```

Representative response:

```text
HTTP/1.1 200 OK
Content-Type: text/plain; charset=utf-8
Date: Wed, 18 Mar 2026 12:04:09 GMT
Content-Length: 13

Hello, world!
```

The timer service uses `build_tick_message()` and emits exactly:

- `Timer trigger executed.`

## Cleanup

```bash
az group delete --name "$HTTP_RG" --yes --no-wait
az group delete --name "$TIMER_RG" --yes --no-wait
```

Representative output:

```text
{"status":"Accepted"}
{"status":"Accepted"}
```
